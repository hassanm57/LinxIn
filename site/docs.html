LinxIn docs

Your AI-run war room for the job hunt. Local-first, human-in-the-loop, built for Pakistan and remote-anywhere.

What is LinxIn

LinxIn turns your AI coding CLI (Claude Code and friends) into a private job-search system that runs entirely on your machine. It reads listings, scores them on Fit, Reality and Trust, flags scams and ghost jobs, and tailors your CV, cover letter and freelance proposals. It never applies for you: AI evaluates and drafts, you decide and submit.

Quick start

LinxIn is a set of skills plus a few small Node scripts. Clone it, install once, and drive it from your AI CLI.

# 1. get it
git clone https://github.com/hassanm57/LinxIn
cd LinxIn
npm install

# 2. onboard (in your AI CLI, e.g. Claude Code)
"set up my profile"

# 3. bring in jobs
"ingest this listing: <paste>"      # any board
npm run scan -- --keywords "backend" --remote-only   # ATS portals

# 4. evaluate → get the three scores + a report
"evaluate the new jobs"

# 5. for anything worth it
"tailor my CV and draft a cover letter for <job>"

# 6. browse it all
npm run dashboard        # http://localhost:4173
Works the same in Claude Code, Codex, OpenCode, Gemini CLI, Qwen, or Copilot CLI — the skills are plain markdown, so the CLI you launch is the only thing that changes.

The three-score model

Every job gets three independent 0–100 scores. A role is worth pursuing only if it clears your thresholds on all three.

  • Fit — should you want it? Skills and stack match, seniority, mission, growth, company health, and comp vs. your target.
  • Reality — can a you-in-Pakistan land it and live on it? Geo-eligibility (the near-veto), timezone overlap, payment feasibility (Payoneer / Wise / Deel / EOR), and pay in PKR purchasing power.
  • Trust — is it legit? Verifiable company and recruiter, minus deductions for scam and ghost-job signals: upfront fees, WhatsApp-only recruiters, CNIC/bank asks before an offer, MLM framing, reposted-forever listings.

Defaults: Fit ≥ 65, Reality ≥ 50, Trust ≥ 70. A Trust score below 40 is quarantined as a likely scam. Every threshold and weight is yours to tune — see Configuration.

The skills

Thirteen focused skills, each a plain-markdown instruction set your AI CLI reads. Intake and writing skills feed the orchestrator; the orchestrator feeds the tracker.

profileOnboarding + your two CV bases (international, PII-stripped, and local).
ingestPaste any listing → a clean structured record in your pipeline.
scanPull fresh listings from Greenhouse / Ashby / Lever portals.
evaluateThe orchestrator: runs the three scores and writes a decision report.
geo-checkAre you actually eligible to be hired from Pakistan?
comp-realityWhat the pay is really worth in PKR, and how you'd get paid.
trust-checkScam and ghost-job detection, tuned for local fraud.
tailor-cvATS-clean CV tailored to one role → PDF.
cover-letterResearched letter that asks you for the human bits → PDF.
proposalFreelance track for Upwork / Fiverr / Toptal, with off-platform scam checks.
interview-prepLikely questions, company research, your best stories.
story-bankReusable STAR + Reflection stories the writing skills draw from.
trackKeeps the pipeline clean: dedup, status, integrity.

Architecture

LinxIn keeps a strict boundary between judgment and mechanics. Anything that must be exact — fetching a job board, rendering a PDF, parsing an id — lives in a Node script. Anything that requires judgment — scoring, drafting, deciding — lives in a skill: a plain-markdown instruction set your AI CLI reads. That split keeps the AI out of the business of ever inventing a salary or a job id.

skills/The 13 markdown skills listed above.
scripts/Deterministic Node tools — scan.mjs (ATS fetch), render-pdf.mjs (Playwright), lib/ats.mjs (Greenhouse/Ashby/Lever adapters).
config/settings.yaml (thresholds, locale, comp targets, FX) and weights.yaml (scoring weights).
profile/Your profile.md, dual CV bases, and stories.md. Gitignored.
data/tracker.md (the pipeline) and reports/ (full evaluation write-ups). Gitignored.
sources/companies.yaml — the ATS boards scan pulls from, plus a Pakistan reference list.
templates/HTML templates for the CV, cover letter, and evaluation report.
output/Generated PDFs. Gitignored.
dashboard/The local web dashboard — a small Node server plus vanilla HTML/CSS/JS.

Configuration

Two files under config/ tune how LinxIn judges a job — nothing about scoring is hardcoded.

  • settings.yaml — your timezone, cost-of-living baseline, comp targets, FX rate, and the Fit/Reality/Trust pass thresholds.
  • weights.yaml — the per-dimension weights behind the Fit and Reality scores. Care more about comp than growth, or geo-eligibility more than timezone? Change the number.

Change a value, re-run evaluate — the next report reflects it immediately.

Scanning sources

sources/companies.yaml lists the companies scan pulls from, grouped by ATS (Greenhouse, Ashby, Lever) with a verified token per company. Add your own — the token is the slug from the company's public job-board URL. scan skips any token that 404s and tells you which one, so a stale entry never breaks a run.

Most companies — especially in Pakistan — don't run a public Greenhouse/Ashby/Lever board. For those, the same file keeps a pakistan_reference list of real, verified career-page URLs. Open one, copy a listing, and ingest it instead — it gets scored and tracked exactly like a scanned job.

The dashboard

npm run dashboard starts a small local web server at http://localhost:4173 — your whole pipeline as a card grid, animated Fit/Reality/Trust rings, and a pursue/skip/scam verdict per role. Click any card to read the full evaluation report in a drawer.

It binds to 127.0.0.1 only and reads data/tracker.md directly — nothing is uploaded, and no other machine on your network can reach it.

Privacy

Everything is local. Your profile, CVs and pipeline are plain-text files on your machine and are gitignored by default. The dashboard binds to 127.0.0.1 and never phones home. Nothing leaves your computer except the applications you choose to submit.

Troubleshooting

  • "Playwright isn't installed" when tailoring a CV → npm install playwright && npx playwright install chromium. The HTML output still works meanwhile; print it to PDF from any browser.
  • scan returns 0 jobs or errors on every board → check that the tokens in sources/companies.yaml haven't gone stale; companies do migrate ATS platforms.
  • Dashboard shows demo data → your data/tracker.md has no jobs yet. Run ingest or scan first.
  • A score looks off → scores are guidance, not verdicts. Adjust config/weights.yaml and config/settings.yaml to match your priorities, then re-run evaluate.
